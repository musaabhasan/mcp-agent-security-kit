# Security Policy

## Reporting Security Issues

Please do not open a public issue for vulnerabilities that could expose users. Use GitHub private vulnerability reporting if enabled, or contact the maintainer through the profile links.

## Scope

This repository provides defensive security tooling and documentation for MCP and agentic AI systems.

Out of scope:

- requests to bypass authentication,
- exploit development against third-party systems,
- real secrets or credentials,
- attacks against systems you do not own or have permission to test.

## Handling Test Data

Use only synthetic examples in issues and pull requests. Redact tokens, internal domains, IP addresses, tenant identifiers, and customer data.
